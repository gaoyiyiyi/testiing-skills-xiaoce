# 用例 Schema

测试用例使用 UTF-8 JSON 编写。根对象表示一份完整的脑图。

## 根对象

必填字段：

- `title`：字符串。作为脑图根节点标题。
- `modules`：非空数组。包含一个或多个模块对象。

## 模块对象

必填字段：

- `name`：字符串。表示产品区域、页面、功能、接口域或业务流程。
- `cases`：非空数组。包含一个或多个用例对象。

## 用例对象

必填字段：

- `title`：字符串。简洁、可读的用例标题。
- `precondition`：字符串。该用例的一组前置条件。
- `steps`：非空数组。包含一个或多个步骤对象。

## 步骤对象

必填字段：

- `action`：字符串。一个具体操作或检查动作。
- `expected`：字符串。该动作唯一对应的预期结果。

## 强规则

- 每条用例必须且只能有一个 `precondition` 字段。
- 每条用例必须至少有一个步骤。
- 每个步骤必须有一个 `action` 和一个 `expected`。
- 如果多个预期结果可以拆分到不同步骤，不要把它们合并成一段模糊描述。
- 不要生成空占位用例。
- 用例标题要稳定、可读；除非源材料要求，否则不要在标题里手动编号。
- XMind 导出模板只展示模块、用例标题、前置条件、步骤和预期结果，不展示来源参考、用例属性、备注或其他元信息。
